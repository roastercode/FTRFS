Item~1 replaces the per-inode opt-in flag with the unconditional
\texttt{INODE\_UNIVERSAL} scheme; the deprecated flag bit is
preserved for v0.1.0/v0.2.0 image compatibility (per axis~3.3).
Item~2 reuses the same \texttt{lib/reed\_solomon} machinery as the
inode and bitmap paths and places parity at the trailing edge of
the 4096-byte block to keep new feature additions local to the
central padding region. Item~3 refines
\texttt{ftrfs\_rs\_decode}'s return convention to yield the symbol
count on success; this count is the seed of the future entropy
computation in format~v4 (per axis~3.2). Item~4a refactors the
superblock serialization paths to use \texttt{offsetof()} throughout
with \texttt{BUILD\_BUG\_ON} sentinels; the 24-byte size sentinel
of \texttt{struct ftrfs\_rs\_event} placed in this commit is the
compile-time tripwire for the format~v4 transition. Item~4b-dirent
fixes a regression where directory entry deletion zeroed
\texttt{d\_rec\_len}, causing iteration loops to misclassify freed
slots as end-of-block; the fix performs a full block scan and
identifies free slots by \texttt{d\_ino == 0}, with a static
invariant guard added to the pre-commit hook.

The format v4 design (item~4 deferred) extends
\texttt{struct ftrfs\_rs\_event} from 24 to 40 bytes with an
entropy field, structural sentinels and per-record CRC; it
migrates the superblock layout to 13~RS subblocks
(\texttt{FTRFS\_SB\_RS\_PARITY\_BYTES} 128~$\to$~208) and provides
strict version reject of v3 images on a v4 kernel.
